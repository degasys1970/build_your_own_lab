What Tools Should You Use in Your Lab?
Module 10 is dedicated to penetration testing tools. Therefore, this section only scratches the surface. Basically, the tools you use in penetration testing depend on the type of testing you are doing. If you are doing testing on a customer environment, you will likely be evaluating various attack surfaces – such as network infrastructure, wireless infrastructure, web servers, database servers, Windows systems, or Linux systems, for example.

Network infrastructure-based tools might include tools for sniffing or manipulating traffic, flooding network devices, and bypassing firewalls and IPSs. For wireless testing purposes, you might use tools for cracking wireless encryption, de-authorizing network devices, and performing on-path attacks (also called man-in-the-middle attacks).

When testing web applications and services, you can find a number of automated tools built specifically for scanning and detecting web vulnerabilities, as well as manual testing tools such as interception proxies. Some of these same tools can be used to test for database vulnerabilities (such as SQL injection vulnerabilities).

For testing the server and client platforms in an environment, you can use a number of automated vulnerability scanning tools to identify things such as outdated software and misconfigurations. With a lot of development targeting mobile platforms, there is an increasing need for testing these applications and the servers that support them. For such testing, you need another set of tools specific to testing mobile applications and the back-end APIs that they typically communicate with. And you should not forget about fuzzing tools, which are normally used for testing the robustness of protocols.

Milyen eszközöket használj a laborodban?
A 10. modul a behatolási tesztelés eszközeire összpontosít. Ezért ez a rész csak a felszínt kapargatja. Alapvetően a behatolási tesztelés során használt eszközök attól függnek, hogy milyen típusú tesztelést végzel. Ha ügyfélkörnyezetben végzel tesztelést, valószínűleg különféle támadási felületeket fogsz értékelni – például hálózati infrastruktúrát, vezeték nélküli infrastruktúrát, webszervereket, adatbázisszervereket, Windows rendszereket vagy Linux rendszereket.

A hálózati infrastruktúra-alapú eszközök közé tartozhatnak a forgalom lehallgatására vagy manipulálására, hálózati eszközök túlterhelésére, valamint tűzfalak és IPS-ek (támadásmegelőző rendszerek) megkerülésére szolgáló eszközök. Vezeték nélküli tesztelési célokra használhatsz olyan eszközöket, amelyek vezeték nélküli titkosítás feltörésére, hálózati eszközök letiltására vagy közbeékelődéses támadások (más néven ember a középen támadások) végrehajtására alkalmasak.

Webalkalmazások és webszolgáltatások tesztelésekor számos automatikus eszközt találhatsz, amelyeket kifejezetten webes sebezhetőségek szkennelésére és észlelésére terveztek, valamint manuális tesztelési eszközöket, például elfogó proxykat. Ugyanezek az eszközök használhatók adatbázis-sebezhetőségek, például SQL injection sebezhetőségek tesztelésére is.

Ha egy környezet szerver- és kliensplatformjait teszteled, számos automatikus sebezhetőségi szkennelő eszközt használhatsz, amelyek segítségével azonosíthatók például elavult szoftverek és hibás konfigurációk. Mivel egyre több fejlesztés irányul mobilplatformokra, egyre nagyobb szükség van ezeknek az alkalmazásoknak és az őket támogató szervereknek a tesztelésére. Ehhez olyan eszközök kellenek, amelyek kifejezetten mobilalkalmazások és az általában azokkal kommunikáló háttérrendszerek API-jainak tesztelésére alkalmasak. Emellett ne feledkezz meg a protokollok robusztusságát tesztelő "fuzzing" eszközökről sem.
